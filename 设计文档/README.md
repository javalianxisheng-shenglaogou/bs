# 设计文档目录

本目录包含多站点CMS系统的所有设计文档、原型图、数据库设计等资料。

## 📁 目录结构

```
设计文档/
├── 原型图/              # UI原型设计图
├── 功能结构图/          # 系统功能结构和模块划分图
├── 数据库设计/          # 数据库设计文档和ER图
└── 需求分析/            # 需求分析文档和用例图
```

## 📋 各目录说明

### 1. 原型图 (prototypes)
存放系统各个页面的UI原型设计图，包括：
- 登录页面原型
- 用户管理页面原型
- 内容管理页面原型
- 站点管理页面原型
- 其他功能页面原型

**支持格式**：`.png`, `.jpg`, `.pdf`, `.rp`（Axure文件）, `.sketch`, `.xd`, `.fig`

### 2. 功能结构图 (function-structure)
存放系统功能结构图和模块划分图，包括：
- 系统整体功能结构图
- 各模块功能划分图
- 系统架构图
- 技术架构图

**支持格式**：`.png`, `.jpg`, `.pdf`, `.drawio`, `.vsdx`, `.xmind`

### 3. 数据库设计 (database-design)
存放数据库设计相关文档，包括：
- 数据字典（表结构和字段说明）
- ER图（实体关系图）
- 逻辑结构设计文档
- 物理结构设计文档
- 索引设计文档

**支持格式**：`.xlsx`, `.md`, `.pdf`, `.png`, `.drawio`

### 4. 需求分析 (requirements)
存放需求分析相关文档，包括：
- 需求分析文档
- 用例图
- 业务流程图
- 功能需求列表
- 非功能需求说明

**支持格式**：`.md`, `.pdf`, `.docx`, `.png`, `.drawio`

## 📝 文件命名规范

### 原型图命名
- 格式：`页面名称-原型图-版本号.扩展名`
- 示例：
  - `登录页面-原型图-v1.0.png`
  - `用户管理-原型图-v2.0.png`
  - `内容编辑-原型图-v1.1.png`

### 功能结构图命名
- 格式：`图表类型-版本号.扩展名`
- 示例：
  - `系统功能结构图-v1.0.png`
  - `模块划分图-v1.0.png`
  - `技术架构图-v2.0.png`

### 数据库设计文件命名
- 格式：`文档类型-版本号.扩展名`
- 示例：
  - `数据字典-v1.0.xlsx`
  - `ER图-v1.0.png`
  - `逻辑结构设计-v1.0.md`

### 需求分析文件命名
- 格式：`文档类型-版本号.扩展名`
- 示例：
  - `需求分析文档-v1.0.md`
  - `用例图-v1.0.png`
  - `业务流程图-v1.0.png`

## 🔄 版本管理

所有设计文档都应该包含版本号，版本号格式为：`vX.Y`
- **X**：主版本号，重大变更时递增
- **Y**：次版本号，小的修改或补充时递增

示例：
- `v1.0` - 初始版本
- `v1.1` - 小的修改
- `v2.0` - 重大变更

## 📌 注意事项

1. **文件格式**：
   - 优先使用可编辑的源文件格式（如 `.drawio`, `.rp`, `.sketch`）
   - 同时提供导出的图片格式（`.png`, `.jpg`）方便查看
   - 文档优先使用 Markdown 格式（`.md`），便于版本控制

2. **文件大小**：
   - 单个文件不超过 10MB
   - 图片建议使用 PNG 格式，分辨率适中
   - 大文件考虑使用压缩或分割

3. **更新记录**：
   - 每次更新文档时，在对应目录的 README 中记录更新日志
   - 保留历史版本，不要直接覆盖

4. **协作规范**：
   - 修改文档前先拉取最新版本
   - 重大修改建议创建新版本
   - 添加清晰的修改说明
