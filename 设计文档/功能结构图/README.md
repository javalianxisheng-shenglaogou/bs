# 功能结构图目录

本目录存放多站点CMS系统的功能结构图、模块划分图和系统架构图。

## 📋 目录说明

本目录包含系统的整体功能结构、模块划分、技术架构等图表，用于展示系统的组织结构和技术实现。

## 📁 文件组织

### 推荐的文件结构

```
功能结构图/
├── 系统功能结构图-v1.0.png          # 系统整体功能结构
├── 模块划分图-v1.0.png              # 各模块功能划分
├── 系统架构图-v1.0.png              # 系统技术架构
├── 前端架构图-v1.0.png              # 前端技术架构
├── 后端架构图-v1.0.png              # 后端技术架构
├── 数据流图-v1.0.png                # 系统数据流向
├── 部署架构图-v1.0.png              # 系统部署架构
└── 源文件/                          # 可编辑的源文件
    ├── 系统功能结构图-v1.0.drawio
    ├── 模块划分图-v1.0.drawio
    └── ...
```

## 📝 文件命名规范

### 命名格式
```
图表类型-版本号.扩展名
```

### 命名示例
- `系统功能结构图-v1.0.png`
- `模块划分图-v2.0.png`
- `技术架构图-v1.1.png`

### 命名规则
1. **图表类型**：使用中文，清晰描述图表内容
2. **版本号**：格式为 `vX.Y`
3. **扩展名**：根据文件格式使用对应扩展名

## 🎨 支持的文件格式

### 源文件格式（优先）
- `.drawio` - Draw.io 文件（推荐，免费开源）
- `.vsdx` - Microsoft Visio 文件
- `.xmind` - XMind 思维导图文件
- `.mmd` - Mermaid 图表文件（文本格式）
- `.puml` - PlantUML 文件（文本格式）

### 导出格式（必需）
- `.png` - PNG 图片（推荐，高清晰度）
- `.jpg` - JPEG 图片
- `.pdf` - PDF 文档（适合打印）
- `.svg` - SVG 矢量图（可缩放）

### 建议
- 同时保存源文件和导出的图片
- 源文件存放在 `源文件/` 子目录
- 导出的图片放在根目录，便于快速查看

## 📐 设计规范

### 画布尺寸
- **标准尺寸**：1920 x 1080 px 或 A4 纸张尺寸
- **宽屏尺寸**：2560 x 1440 px（适合大屏展示）
- **打印尺寸**：A3 或 A4（300 DPI）

### 导出设置
- **分辨率**：至少 150 DPI，推荐 300 DPI
- **格式**：PNG（透明背景）或 PDF
- **压缩**：适度压缩，保持清晰度

### 设计要点
- 清晰的层级结构
- 统一的图形样式
- 合理的颜色搭配
- 清晰的文字标注
- 适当的留白空间

## 📊 图表类型说明

### 1. 系统功能结构图
**用途**：展示系统的整体功能模块和层级关系

**内容包括**：
- 系统主要功能模块
- 各模块的子功能
- 模块之间的关系
- 功能的层级结构

**示例结构**：
```
多站点CMS系统
├── 用户管理
│   ├── 用户列表
│   ├── 角色管理
│   └── 权限管理
├── 站点管理
│   ├── 站点列表
│   └── 站点配置
├── 内容管理
│   ├── 内容列表
│   ├── 内容编辑
│   └── 版本控制
└── ...
```

### 2. 模块划分图
**用途**：展示系统的模块划分和模块间的依赖关系

**内容包括**：
- 前端模块
- 后端模块
- 数据库模块
- 第三方服务模块
- 模块间的调用关系

### 3. 系统架构图
**用途**：展示系统的技术架构和技术栈

**内容包括**：
- 前端技术栈
- 后端技术栈
- 数据库技术
- 中间件和工具
- 部署环境

**示例层次**：
```
┌─────────────────────────────────────┐
│         前端层（Frontend）           │
│  Vue 3 + TypeScript + Element Plus  │
└─────────────────────────────────────┘
              ↓ HTTP/HTTPS
┌─────────────────────────────────────┐
│         后端层（Backend）            │
│  Spring Boot + Spring Security      │
└─────────────────────────────────────┘
              ↓ JDBC
┌─────────────────────────────────────┐
│         数据层（Database）           │
│         MySQL 8.0                   │
└─────────────────────────────────────┘
```

### 4. 前端架构图
**用途**：详细展示前端的技术架构

**内容包括**：
- 组件结构
- 状态管理
- 路由配置
- API调用
- 工具库

### 5. 后端架构图
**用途**：详细展示后端的技术架构

**内容包括**：
- 分层架构（Controller、Service、Repository）
- 安全认证（JWT、Spring Security）
- 数据访问（JPA、Hibernate）
- 缓存策略
- 日志系统

### 6. 数据流图
**用途**：展示系统中数据的流向和处理过程

**内容包括**：
- 用户请求流程
- 数据处理流程
- 数据存储流程
- 数据返回流程

### 7. 部署架构图
**用途**：展示系统的部署架构和服务器配置

**内容包括**：
- 服务器配置
- 负载均衡
- 数据库集群
- 缓存服务
- 文件存储

## 🎯 设计建议

### 颜色方案
- **主色调**：蓝色系（专业、稳重）
- **辅助色**：绿色（成功）、红色（警告）、黄色（提示）
- **背景色**：白色或浅灰色
- **文字色**：深灰色或黑色

### 图形样式
- **矩形**：表示模块或组件
- **圆角矩形**：表示页面或界面
- **圆形**：表示开始或结束
- **菱形**：表示判断或选择
- **箭头**：表示流程或关系

### 布局建议
- **从上到下**：表示层级关系
- **从左到右**：表示流程关系
- **居中对齐**：保持整洁美观
- **适当留白**：避免拥挤
