## 📝 文件命名规范

### 命名格式
```
页面名称-原型图-版本号.扩展名
```

### 命名示例
- `登录页面-原型图-v1.0.png`
- `用户管理-原型图-v2.0.png`
- `内容编辑-原型图-v1.1.png`

### 命名规则
1. **页面名称**：使用中文，简洁明了
2. **固定标识**：统一使用 `-原型图-`
3. **版本号**：格式为 `vX.Y`
4. **扩展名**：根据文件格式使用对应扩展名

## 🎨 支持的文件格式

### 源文件格式（优先）
- `.rp` - Axure RP 文件
- `.sketch` - Sketch 文件
- `.xd` - Adobe XD 文件
- `.fig` - Figma 文件（导出）
- `.psd` - Photoshop 文件

### 导出格式（必需）
- `.png` - PNG 图片（推荐，透明背景）
- `.jpg` - JPEG 图片
- `.pdf` - PDF 文档（适合打印）

### 建议
- 同时保存源文件和导出的图片
- 源文件便于后续修改
- 图片文件便于快速查看和分享

## 📐 设计规范

### 画布尺寸
- **PC端**：1920 x 1080 px（标准桌面分辨率）
- **移动端**：375 x 812 px（iPhone X 尺寸）
- **平板端**：768 x 1024 px（iPad 尺寸）

### 导出设置
- **分辨率**：72 DPI（屏幕显示）或 300 DPI（打印）
- **格式**：PNG（带透明背景）
- **压缩**：适度压缩，保持清晰度

### 标注要求
- 关键尺寸标注
- 颜色值标注（HEX 或 RGB）
- 字体和字号标注
- 交互说明标注

## 🎯 设计要点

### 1. 一致性
- 保持所有页面的设计风格一致
- 使用统一的颜色方案
- 使用统一的字体和字号
- 使用统一的组件样式

### 2. 易用性
- 清晰的信息层级
- 合理的操作流程
- 明确的操作反馈
- 友好的错误提示

### 3. 响应式设计
- 考虑不同屏幕尺寸的适配
- 移动端优先或桌面端优先
- 关键功能在小屏幕上的可用性

### 4. 无障碍设计
- 足够的颜色对比度
- 清晰的文字大小
- 键盘导航支持
- 屏幕阅读器友好

   - 严格遵循命名规范
   - 避免使用特殊字符
   - 使用有意义的名称
